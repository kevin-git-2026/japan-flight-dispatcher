# ✈️ 日本航班智能搜索与规划 · Japan Flight Dispatcher

> 为 X-Plane 11/12（兼容 MSFS）的飞行模拟玩家自动生成「日本地区」飞行任务：智能抽取或指定起降机场 → 查询机场间的官方 AIP 推荐航路 → 交叉比对现实世界航班排班 → 输出一份可直接用于模拟飞行的航班规划单。

**当前版本：v1.4.0** ｜ 平台：Windows（兼容类 Unix）｜ 形态：纯图形界面桌面程序（PyInstaller 打包的 `flight_dispatcher.exe`）

---

## ✨ 主要功能

| 功能 | 说明 |
| --- | --- |
| 🎲 起降机场抽取 | 完全随机、固定出发 / 固定目的、或两者均固定 |
| 🛣️ AIP 推荐航路查询 | 基于 jp-routes，按起降 ICAO 精确匹配官方推荐航路 |
| 🛂 现实航班排班查询 | 基于 FlightAware，可按航司 / 机型 / 起飞时间筛选 |
| 🪪 模拟呼号生成 | 查不到真实排班时，按航线两端所在大区挑选合理航司生成模拟呼号 |
| 🗺️ 多源地景检测 | 扫描 X-Plane `Custom Scenery` 与 MSFS `Community`，标注机场地景来源（XP / MSFS） |
| 🛡️ 军用机场标记 | 标注军用机场并优先抽取民用航线（避免随到可能缺民航设施的机场） |
| 🛬 跑道长度过滤 | 按机型所需最短跑道长度筛选可用机场 |
| 📏 航程范围控制 | 按大圆距离（NM）上下限筛选航线 |
| 🧭 **本地航路生成**（v1.4.0） | 查不到官方 AIP 航路时，用自带导航数据本地 A\* 算一条参考航路（标注非官方、仅供参考） |
| 🛩️ **SimBrief 一键派遣**（v1.4.0） | 结果卡链接，用您自己浏览器的 SimBrief 登录态出专业飞行计划（OFP），无需任何凭据 |
| ✈️ **机型可搜索下拉**（v1.4.0） | 从 212 种 SimBrief 机型库中搜索 / 选择 |
| 🗺️ **航路交互地图**（v1.4.0） | 每条航路在真实地图（OpenStreetMap）上查看，可拖拽缩放 |
| 📐 **航路距离与偏差**（v1.4.0） | 每条航路显示沿真实航路的精确长度与相对大圆的绕行 +X% |
| 🌟 Volanta 优先未飞航线 | 读取您的 Volanta 已飞记录，随机规划时软优先您还没飞过的航线 |

## 📥 下载与安装

1. 前往 **[Releases](https://github.com/kevin-git-2026/japan-flight-dispatcher/releases)** 下载最新版压缩包。
2. 解压后，请保持以下文件 / 文件夹在**同一目录**：
   - `flight_dispatcher.exe`
   - `NavData/`（导航数据，详见下方说明）
   - `aircrafts.json`（机型库）
3. 双击 `flight_dispatcher.exe` 即可打开窗口。

> 💡 程序首次运行会在同目录自动生成 `installed_scenery.json`、`airlines.json`、`routes_cache.csv` 等缓存 / 配置文件，属正常现象。

## 🚀 使用

打开后：左侧填写规划条件（**留空＝随机 / 默认**），点 **「🛫 规划航线」**，结果显示在右侧航线卡，底部是运行日志；Volanta 同步、「仅在有地景的机场间规划」等都在界面上点选。

| 输入项 | 示例 | 留空行为 |
| --- | --- | --- |
| 出发 ICAO | `RJAA` | 随机 |
| 目的 ICAO | `RJBB` | 随机 |
| 执飞航司 | `ANA` | 不限 |
| 机型 | `737`（可搜索下拉） | 跳过 |
| 时间区间 | `08:00-15:30` | 跳过 |
| 最短跑道 | `1800m` 或 `5900ft` | 5900 ft |
| 航程范围 | `200` — `450`（NM） | 200 / 450 |

## ⚠️ 使用前请注意

### 1. 导航数据（NavData）

- 程序根目录自带一个 `NavData` 文件夹，直接读取里面的导航数据，**不再强制依赖 X-Plane 安装目录**——即使您只飞 MSFS、根本没装 X-Plane 也能用。
- 请保证 `NavData` 文件夹与 `flight_dispatcher.exe` 放在同一目录下。
- 导航数据有有效期（AIRAC 周期，约 28 天）。每次启动会自检是否过期，过期会提示您前往 **[Navigraph 下载页](https://navigraph.com/downloads)** 重新下载「X-Plane 12」的导航数据，用新数据替换 `NavData` 文件夹后重启即可。

### 2. 联网

- 查询 AIP 航路（jp-routes）、现实航班排班（FlightAware）、加载交互地图瓦片均需联网。
- 首次联网后会生成 AIP 航路缓存，之后可离线查询 AIP 航路；但查询现实航班始终需要联网。

### 3. 地景检测

- 自动扫描 X-Plane 的 `Custom Scenery` 与 MSFS 的 `Community` 文件夹，标注机场地景来源（`[地景:XP]` / `[地景:MSFS]` / `[地景:XP+MSFS]`，未装则标 `[⚠️无地景]`）。
- 结果缓存到 `installed_scenery.json`，目录未变则下次启动秒开；新增 / 删除 / 替换地景包会自动重扫。
- 没找到任何模拟器地景目录时自动跳过（不标注地景），不影响正常规划。
- 随机规划时可勾选「仅在两端都已安装地景的机场之间规划」，配合 Volanta 优先未飞，专挑您装了地景、又还没飞过的航线。

### 4. Volanta 优先未飞航线（可选）

- 如果您用 Volanta 记录飞行，程序会读取您已飞过的航线，在随机规划时**软优先**您还没飞过的航线（不会完全排除已飞航线）。
- 不使用 Volanta 的用户无需任何操作，不影响其它功能。
- 同步方式：程序用您**浏览器里自己的 Volanta 登录会话**自动获取您的全部航班数据（类比 yt-dlp 的「从浏览器读取登录态」）。若本机无有效登录，会打开 `fly.volanta.app/map` 让您登录一下（登录凭据约 14 天有效）。
- **仅支持 Chromium 系浏览器**（Edge / Chrome / Brave）；Firefox / Safari 暂不支持自动获取（跨平台支持是后续重点）。
- **隐私**：登录令牌只用于这一次请求，**绝不保存、绝不发送到任何第三方**。
- 同步偏好、已飞航线累积库、最后获取时间统一保存在 `volanta_data.json`。

### 5. 机型库 / 交互地图 / SimBrief（v1.4.0）

- 机型库 `aircrafts.json` 请与 `flight_dispatcher.exe` 放在同一目录（已随程序分发）；机型输入框可直接搜索 / 下拉选择。
- 航路交互地图：结果卡每条航路有「🗺️」链接，点开在真实地图上画出该航路。地图用到第三方库（已打包进 exe）；万一运行环境缺该库，只是地图链接不显示，其它功能照常。
- SimBrief 一键派遣：点结果卡上的「🛩️ SimBrief 一键派遣」链接，用您自己浏览器里已登录的 SimBrief 出专业飞行计划（OFP）——无需填写任何账号凭据。

## ⚙️ 技术说明

- 主体为 Python 标准库实现；v1.4.0 起交互地图引入第三方库 `tkintermapview` + `Pillow`（缺失时地图自动禁用，其它仍纯标准库可用）。
- 外部数据源为两个公开网站（jp-routes、FlightAware），均为页面抓取而非官方 API，可能随对方改版而失效。
- 全部用户界面文案为简体中文。

> ⚠️ 本工具仅用于**飞行模拟**用途，所生成的航路 / 排班**不可用于真实航空运行**。
