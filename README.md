# ✈️ 日本航班智能搜索与规划 · Japan Flight Dispatcher

> 为 X-Plane 11/12（兼容 MSFS）的飞行模拟玩家自动生成「日本地区」飞行任务：智能抽取或指定起降机场 → 查询机场间的官方 AIP 推荐航路 → 交叉比对现实世界航班排班 → 输出一份可直接用于模拟飞行的航班规划单。

**当前版本：v1.6.0** ｜ 平台：Windows（兼容类 Unix）｜ 形态：纯图形界面桌面程序（PyInstaller 打包的 `flight_dispatcher.exe`）

---

## ✨ 主要功能

| 功能 | 说明 |
| --- | --- |
| 🎲 起降机场抽取 | 完全随机、固定出发 / 固定目的、或两者均固定 |
| 🛣️ AIP 推荐航路查询 | 基于 jp-routes，按起降 ICAO 精确匹配官方推荐航路 |
| 🛂 现实航班排班查询 | 基于 FlightAware，可按航司 / 机型 / 起飞时间筛选 |
| 🪪 模拟呼号生成 | 查不到真实排班时，按航线两端所在大区挑选合理航司生成模拟呼号 |
| 🗺️ 多源地景检测 | 扫描 X-Plane `Custom Scenery` 与 MSFS `Community`，标注机场地景来源（XP / MSFS） |
| 🛡️ 军用机场标记 | 标注军用机场并优先抽取民用航线（避免随到可能缺民航设施的机场） |
| 🛬 跑道长度过滤 | 按机型所需最短跑道长度筛选可用机场 |
| 📏 航程范围控制 | 按大圆距离（NM）上下限筛选航线 |
| 🧭 **本地航路生成**（v1.4.0；v1.4.1 增强） | 查不到官方 AIP 航路时，用自带导航数据本地 A\* 算一条参考航路——优先 RNAV 航路、沿真实高频「空中走廊」、用各机场真实进/离场端点，更贴近真实运行（标注非官方、仅供参考） |
| 🛩️ **SimBrief 一键派遣**（v1.4.0） | 结果卡链接，用您自己浏览器的 SimBrief 登录态出专业飞行计划（OFP），无需任何凭据 |
| ✈️ **机型可搜索下拉**（v1.4.0） | 从 212 种 SimBrief 机型库中搜索 / 选择 |
| 🗺️ **航路交互地图**（v1.4.0） | 每条航路在真实地图（OpenStreetMap）上查看，可拖拽缩放 |
| 📐 **航路距离与偏差**（v1.4.0） | 每条航路显示沿真实航路的精确长度与相对大圆的绕行 +X% |
| 🛫 **跑道 + 进离场程序 + 天气**（v1.4.1；v1.5.0 增强） | 规划后为航班选出发 / 到达跑道与 SID/STAR，配 METAR/TAF 天气算每条跑道的逆 / 侧风与适航；**小机场 METAR 缺测 / 过期时，自动用 Open-Meteo 网格天气合成一条参考 METAR**（清楚标注「模型合成·非实测」），并拼进 SimBrief |
| ⚙️ **机场运行规则编辑器**（v1.5.0） | 可视化编写各机场按【时段 + 星期 + 风 + 天气】切换的运行规则（跑道 / SID / STAR / IAP），保存为 `operation.json`；换向门槛支持顺风 / 逆风 / **侧风**（如都心「16 侧风超即改 22/23」） |
| 🎯 **运行规则自动应用**（v1.6.0） | 规划一次航班后，按当前（或您填的 EOBT）时段 + 星期 + 实测风 + 天气匹配上面编写的规则，**自动为该航班预选跑道 / SID / STAR / IAP** 并标注依据；决策支持，您随时可改选（也可一键关闭）。该 EOBT 同时提交给 SimBrief |
| 🌟 Volanta 优先未飞航线 | 读取您的 Volanta 已飞记录，随机规划时软优先您还没飞过的航线 |

## 📥 下载与安装

1. 前往 **[Releases](https://github.com/kevin-git-2026/japan-flight-dispatcher/releases)** 下载最新版压缩包。
2. 解压后，请保持以下文件 / 文件夹在**同一目录**：
   - `flight_dispatcher.exe`
   - `NavData/`（导航数据，详见下方说明）
   - `aircrafts.json`（机型库）
   - `transfer_points.json`（进离场移管点）
   - `operation.json`（机场运行规则，**已附带一套完整的羽田 RJTT 规则作样例**；可用「⚙️ 编辑机场运行规则」照着改写其它机场，删掉该文件即为空规则）
3. 双击 `flight_dispatcher.exe` 即可打开窗口。

> 💡 程序首次运行会在同目录自动生成 `installed_scenery.json`、`airlines.json`、`routes_cache.csv` 等缓存 / 配置文件，属正常现象。

## 🚀 使用

打开后：左侧填写规划条件（**留空＝随机 / 默认**），点 **「🛫 规划航线」**，结果显示在右侧航线卡，底部是运行日志；Volanta 同步、「仅在有地景的机场间规划」等都在界面上点选。

| 输入项 | 示例 | 留空行为 |
| --- | --- | --- |
| 出发 ICAO | `RJAA` | 随机 |
| 目的 ICAO | `RJBB` | 随机 |
| 执飞航司 | `ANA` | 不限 |
| 机型 | `737`（可搜索下拉） | 跳过 |
| 时间区间 | `08:00-15:30` | 跳过 |
| 最短跑道 | `1800m` 或 `5900ft` | 5900 ft |
| 航程范围 | `200` — `450`（NM） | 200 / 450 |

## ⚠️ 使用前请注意

### 1. 导航数据（NavData）

- 程序根目录自带一个 `NavData` 文件夹，直接读取里面的导航数据，**不再强制依赖 X-Plane 安装目录**——即使您只飞 MSFS、根本没装 X-Plane 也能用。
- 请保证 `NavData` 文件夹与 `flight_dispatcher.exe` 放在同一目录下。
- 导航数据有有效期（AIRAC 周期，约 28 天）。每次启动会自检是否过期，过期会提示您前往 **[Navigraph 下载页](https://navigraph.com/downloads)** 重新下载「X-Plane 12」的导航数据，用新数据替换 `NavData` 文件夹后重启即可。

### 2. 联网

- 查询 AIP 航路（jp-routes）、现实航班排班（FlightAware）、加载交互地图瓦片均需联网。
- 首次联网后会生成 AIP 航路缓存，之后可离线查询 AIP 航路；但查询现实航班始终需要联网。

### 3. 地景检测

- 自动扫描 X-Plane 的 `Custom Scenery` 与 MSFS 的 `Community` 文件夹，标注机场地景来源（`[地景:XP]` / `[地景:MSFS]` / `[地景:XP+MSFS]`，未装则标 `[⚠️无地景]`）。
- 结果缓存到 `installed_scenery.json`，目录未变则下次启动秒开；新增 / 删除 / 替换地景包会自动重扫。
- 没找到任何模拟器地景目录时自动跳过（不标注地景），不影响正常规划。
- 随机规划时可勾选「仅在两端都已安装地景的机场之间规划」，配合 Volanta 优先未飞，专挑您装了地景、又还没飞过的航线。

### 4. Volanta 优先未飞航线（可选）

- 如果您用 Volanta 记录飞行，程序会读取您已飞过的航线，在随机规划时**软优先**您还没飞过的航线（不会完全排除已飞航线）。
- 不使用 Volanta 的用户无需任何操作，不影响其它功能。
- 同步方式：程序用您**浏览器里自己的 Volanta 登录会话**自动获取您的全部航班数据（类比 yt-dlp 的「从浏览器读取登录态」）。若本机无有效登录，会打开 `fly.volanta.app/map` 让您登录一下（登录凭据约 14 天有效）。
- **仅支持 Chromium 系浏览器**（Edge / Chrome / Brave）；Firefox / Safari 暂不支持自动获取（跨平台支持是后续重点）。
- **隐私**：登录令牌只用于这一次请求，**绝不保存、绝不发送到任何第三方**。
- 同步偏好、已飞航线累积库、最后获取时间统一保存在 `volanta_data.json`。

### 5. 机型库 / 交互地图 / SimBrief（v1.4.0）

- 机型库 `aircrafts.json` 请与 `flight_dispatcher.exe` 放在同一目录（已随程序分发）；机型输入框可直接搜索 / 下拉选择。
- 航路交互地图：结果卡每条航路有「🗺️」链接，点开在真实地图上画出该航路。地图用到第三方库（已打包进 exe）；万一运行环境缺该库，只是地图链接不显示，其它功能照常。
- SimBrief 一键派遣：点结果卡上的「🛩️ SimBrief 一键派遣」链接，用您自己浏览器里已登录的 SimBrief 出专业飞行计划（OFP）——无需填写任何账号凭据。

## 📋 更新日志

### v1.5.0 → v1.6.0
- 🎯 **运行规则自动应用（规划时按规则预选跑道 / 程序）**：v1.5.0 只能编写并保存运行规则，本版让它们**在规划时真正生效**。规划一次航班后，进离场面板新增 **EOBT（撤轮挡时刻，JST）输入**与**「按机场运行规则预选」开关**：程序按 当前（或您填的）时段 + 星期几 + 实测风 + 天气，从您为该机场编写的规则里匹配出应用哪一条，**自动为该航班预选出发 / 到达跑道、SID / STAR，并展示对应仪表进近（IAP）**，还会标注依据（如「🎯 运行规则：南風C 都心運用 → RW22 / …」）。离场机场按 EOBT、到达机场按预计到达时刻（ETA）匹配。这是**决策支持**——预选后您仍可随意改选，或一键关闭开关回到「按风排序」的原行为。填入的 EOBT 也会一并提交给 SimBrief。
- 🌬️ **换向门槛新增「侧风」**：编辑器的换向门槛除了顺风 / 逆风，新增**侧风（crosswind）**——把「相对某跑道的侧风超过 N 节就换落别的跑道」这类规则（如羽田都心運用：16L/16R 侧风超约 15 节即改落 22/23）变成可视化可编写、任何机场通用。

### v1.4.1 → v1.5.0
- 🌐 **网格天气回退（小机场夜间也有天气）**：很多日本小机场（大岛 RJTO、八丈岛 RJTH、松本 RJAF、壱岐 RJER 等）的 METAR **只在日间更新，夜间只能拿到白天的旧报文、甚至完全取不到**。本版在 METAR **缺测或过期**时，自动用机场坐标处的 **Open-Meteo 网格模型天气**（日本本地 5km 模式）**合成一条标准格式的参考 METAR**，让您夜间飞小机场也能对着风选跑道。合成报文清楚标注「🌐 模型合成·非实测」，能见度从严处理（不会凭空报雾），风 / 温 / 气压很准。天气数据来自 [Open-Meteo](https://open-meteo.com)（CC BY 4.0）。
- ⚙️ **机场运行规则编辑器**：日本很多机场（如羽田）按【时段 + 风】切换跑道 / 进离场程序（南風 / 北風運用、夜間運用等）。新增「⚙️ 编辑机场运行规则」可视化编辑器，让您为任意机场编写这些规则——**运行条件（时段 + 风向扇区 / 风速）、离场跑道 + SID、进场跑道 + STAR + 仪表进近（IAP）**，各项从该机场真实导航数据中选择，保存为 `operation.json`。支持规则的新增 / 编辑 / 删除、多机场分别管理。（本版先做编辑与存储；让这些规则在规划时自动预选跑道 / 程序将在后续版本接入。）

## ⚙️ 技术说明

- 主体为 Python 标准库实现；v1.4.0 起交互地图引入第三方库 `tkintermapview` + `Pillow`（缺失时地图自动禁用，其它仍纯标准库可用）。网格天气（v1.5.0）仍为纯标准库（`urllib`+`json`），无新增第三方依赖。
- 外部数据源：航路（jp-routes）与排班（FlightAware）为页面抓取而非官方 API，可能随对方改版而失效；天气为 NOAA `tgftp`（METAR/TAF）与 **[Open-Meteo](https://open-meteo.com)** 网格模型（缺测 / 过期时的回退）。
- 全部用户界面文案为简体中文。

## 🙏 数据来源与致谢

- 天气模型数据来自 **[Open-Meteo](https://open-meteo.com)**，依 **CC BY 4.0** 授权使用（网格天气回退功能）；实时观测 / 预报报文来自 **NOAA**。
- 导航数据由用户自备的 Navigraph `NavData` 提供；AIP 航路来自 jp-routes 公开数据。

> ⚠️ 本工具仅用于**飞行模拟**用途，所生成的航路 / 排班 / 合成天气**不可用于真实航空运行**。
