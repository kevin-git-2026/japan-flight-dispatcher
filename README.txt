当前版本：v1.3.1

★ v1.3.1 起本程序为纯【图形界面】：双击 flight_dispatcher.exe 即打开窗口——左侧填写规划条件（留空＝随机/默认）、点「🛫 规划航线」，结果显示在右侧航线卡，底部是运行日志；Volanta 同步、「仅在有地景的机场间规划」等都在界面上点选。

使用前请注意！！！

1. 导航数据（NavData）：本程序根目录自带一个 NavData 文件夹，程序会直接读取里面的 earth_aptmeta.dat 导航数据，不再强制依赖 X-Plane 安装目录——即使你只飞 MSFS、根本没装 X-Plane 也能用。
- 请保证 NavData 文件夹与 flight_dispatcher.exe 放在同一目录下。
- 导航数据有有效期（AIRAC 周期，约 28 天）。每次启动程序会自检 NavData 是否过期，过期会提示你前往 Navigraph 下载页 https://navigraph.com/downloads 重新下载「X-Plane 12」的导航数据，用新数据替换 NavData 文件夹里的内容后重启即可。
- （兼容）你也可以把 earth_aptmeta.dat 直接放到 exe 工作目录，或让程序自动扫描 X-Plane 根目录读取，作为后备。

2. 这个脚本目前基于 jp-routes.vercel.app 查询 AIP 航路数据，基于 flightaware 查询日本的航班数据，所以本脚本需要联网。第一次联网后脚本会生成 AIP 航路的缓存数据，之后可以离线使用脚本查询 AIP 航路。但查询现实航班依旧需要联网。

3. 地景检测：脚本会自动扫描 X-Plane 的 Custom Scenery 文件夹与 MSFS 的 Community 文件夹，判断机场是否已经安装了插件地景，并标注来源（[地景:XP] / [地景:MSFS] / [地景:XP+MSFS]，未装则标 [⚠️无地景]）。
- 扫描结果会缓存到 installed_scenery.json，地景目录没有变化时下次启动秒开；新增/删除/替换地景包会自动重扫。
- 不再需要 scenery packs.ini 文件。如果没找到任何模拟器地景目录，地景检测会自动跳过（不标注地景），不影响正常规划。
- （v1.3.0 新）随机规划时可勾选「仅在两端都已安装地景的机场之间规划」，配合 Volanta 优先未飞，专挑你装了地景、又还没飞过的航线。未检测到地景目录时该选项不可用（自动灰显/跳过）。

4. Volanta 优先未飞航线（可选）：如果你使用 Volanta 记录飞行，程序会读取你已经飞过的航线，在随机规划时优先安排你还没飞过的航线（软优先，不会完全排除已飞航线）。
- 不使用 Volanta 的用户无需任何操作，回车跳过即可，不影响其它功能。
- 首次启动会询问是否同步 Volanta 数据：
  · 选 Y：程序用你浏览器里的 Volanta 登录会话自动获取你的全部航班数据（无需开发者工具、不用滚动）。若本机没有有效登录（例如很久没登录、或刚清过浏览器数据/Cookie），会打开 fly.volanta.app/map（地图页）让你登录一下，登录后自动取回（登录凭据约 14 天有效，期间无需再登录；过期后会再次引导你到地图页登录）。并且【从此记住选择，以后启动自动同步、不再询问】。
  · 回车 / N：本次跳过，下次启动再问（随时可改主意，不会被锁死）。
- 仅支持 Chromium 系浏览器（Edge / Chrome / Brave）：程序从这些浏览器读取你自己的 Volanta 登录令牌。Firefox / Safari 等暂不支持自动获取，可改用下面的手动方式（跨平台/非 Chromium 浏览器支持是后续更新的重点）。
- 说明（隐私）：这与 yt-dlp 的“从浏览器读取登录态”同理——程序只读取你自己浏览器里你自己的 Volanta 登录令牌，在本机调用 Volanta 自己的接口下载你自己的飞行数据；该令牌只用于这一次请求，绝不保存、绝不发送到任何第三方。
- 进阶/手动：也可在浏览器开发者工具(F12) → 网络 → 找到 Flights 请求 → 右键 Copy response → 保存为 volanta_flights.json 放到程序目录，程序会读取并合并后自动删除该文件。
- 程序把同步偏好、已飞航线累积库（只增不减）、最后获取时间统一保存在 volanta_data.json 一个文件里，历史数据不会丢失。

脚本功能

1. 自定义或随机抽取起降机场

2. 基于 jp-routes.vercel.app 智能查询起降机场间的推荐航路

3. 基于 flightaware 与自定义机型与起飞时间筛选航班号

4. 模拟航班号生成：查不到真实排班时，按航线两端所在地区挑选合理航司生成模拟航班号（航司可在 airlines.json 自行增删改）

5. 多源地景检测：扫描 X-Plane 与 MSFS 地景文件夹，自动标注机场地景来源（XP / MSFS）

6. 自带 NavData 导航数据，并自检 AIRAC 周期，过期时提示更新

7. Volanta 优先未飞航线：读取已飞记录，随机规划时软优先你没飞过的航线

8. 图形界面（GUI）：表单填条件、一键规划、右侧航线卡 + 底部日志区，Volanta 同步与各开关都在界面操作（v1.3.1 起为唯一界面）

9. 仅地景机场随机规划：随机时可只在两端都装了地景的机场之间抽线，与 Volanta 优先未飞叠加

首次启动询问是否同步，选 Y 后用浏览器登录会话自动获取并记住偏好（以后启动自动同步），同步偏好与已飞累积库统一保存在 volanta_data.json

（避免「北海道航司飞冲绳航线」之类的离谱组合），航司数据放在可编辑的 airlines.json

- 随机抽线优先民用机场：避免抽到可能缺民航设施、不一定可飞的军用机场（你固定某个军用机场时仍可正常规划）
